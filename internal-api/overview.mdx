The internal API is the boundary between Percy's two halves: the **web dashboard** (a Rust backend) and the **bot** (Python). Every change you make on the dashboard is proxied through this API to the bot, which owns the database and its caches.

<Note>
  This API is **internal infrastructure**, not a public API. It listens on localhost and is reached only by Percy's own dashboard backend. It is documented here for operators and contributors. For the public platform API, see the [Klappstuhl.me API](/api-reference/introduction).
</Note>

## Why it exists

The dashboard must **never** write directly to the bot's database — Percy memoizes config in memory, so a direct write would create split-brain state. Instead, every mutation flows through this API, which performs the write **and** invalidates the relevant cache:

```
Browser → Dashboard (Rust BFF) → Percy Internal API → PostgreSQL
                                          │
                                  cache.invalidate()
```

## Base path and host

```
http://127.0.0.1:<INTERNAL_API_PORT>/api/internal
```

The server binds to `127.0.0.1` only. The port is set by `INTERNAL_API_PORT`.

## Authentication

Every `/api/internal/*` request requires a bearer token, enforced globally:

```
Authorization: Bearer <INTERNAL_API_TOKEN>
```

The dashboard and the bot share this pre-shared token (`percy.api_token` on the dashboard, `INTERNAL_API_TOKEN` on the bot). Requests without it are rejected.

<Note>
  Routes under `/api/webhooks/*` are **exempt** from the bearer middleware — they're reached by external bot lists (top.gg, discordbotlist.com), not the dashboard, and validate their own per-service secret instead.
</Note>

## Conventions

- **Cache invalidation is mandatory.** Any handler that mutates guild or user config invalidates the matching cache so the bot and dashboard never disagree.
- **Member actions hit Discord directly.** Endpoints like ban/kick call the Discord API; they don't mutate the database.
- **Read-only endpoints** (stats, browsing) require no invalidation.

See the [Endpoints reference](/internal-api/endpoints) for the full route table.
