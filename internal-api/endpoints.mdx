Every route below requires `Authorization: Bearer <INTERNAL_API_TOKEN>` (except the webhook routes, which validate their own per-service secret). See the [overview](/internal-api/overview) for the auth and caching model.

## Path conventions

Routes are mounted under a versioned prefix and a scope:

| Scope | Base path |
|---|---|
| Guild-scoped | `/api/v1/guilds/{guild_id}` |
| User-scoped | `/api/v1/users/{discord_id}` |
| Root (bot-wide) | `/api/v1` |
| Webhooks (external) | `/api/webhooks` |

Paths in the tables below are **relative to the scope** noted in each section's heading.

<Note>
  The internal API also serves its own interactive reference (Scalar) at `/docs`, generated from the live OpenAPI schema, and sends an `X-API-Version` header on every response.
</Note>
